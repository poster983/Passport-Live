(inner) getPasses()

API: GET /api/passes/

Gets all passes from passport or filters them. JWT Required Only for admins

API Route: GET /api/passes/
Query Params
Name Type Attributes Description
id String <optional>
 ID of the pass itself (will still return an array)
fromPerson String <optional>
 User ID of the person that the migrator is leaving from
toPerson String <optional>
 User ID of the person that the migrator is going to
migrator String <optional>
 User ID of the person that is actually moving
requester String <optional>
 User ID of the person that requested the pass
period String | Array.<String> <optional>
 An array string of period constants. (Feed array like "?period=a&period=b&period=c")
date_from ISOString <optional>
 Lower limit for the date. inclusive
date_to ISOString <optional>
 Upper limit for the date. inclusive
Response
Array of pass objects

Array.<Object>

(async, inner) newPassForMe(req, res, next)

API: POST /api/passes/me/ Async

Creates a new Pass from/for currently logged in account. This one used the JWT to find the requester and migrator. this only allows the signed in person to request a pass for themselves REQUIRES JWT Authorization in headers.

Example

Body Structure (application/json): 

{
   "fromPerson": (id Of Account)
   "toPerson": (id Of Account)
   "migrator"; (id of account)
   "period": (a period constant)
   "date": (an date string in iso standered)
}
